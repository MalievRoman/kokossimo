# Проверка загрузки изображений товаров из МойСклад

Если картинки в каталоге не отображаются (плейсхолдеры или пустые места), по шагам проверьте следующее.

## 1. Диагностика одним запросом

Из папки бэкенда выполните:

```bash
python manage.py check_product_image
```

Без аргумента проверяется первый товар с `moysklad_id`. Для конкретного товара:

```bash
python manage.py check_product_image 123
```

Команда выведет:
- есть ли у товара `external_image_url` и `moysklad_id` в БД;
- подключается ли клиент к API МойСклад;
- есть ли в ответе API поле `images` и ссылки `downloadHref`;
- удаётся ли скачать файл по этой ссылке.

По сообщению на шаге, где всё ломается, можно понять причину.

## 2. Что проверить вручную

### Учётные данные МойСклад

В `.env` должны быть заданы либо `MOYSKLAD_TOKEN`, либо пара `MOYSKLAD_LOGIN` и `MOYSKLAD_PASSWORD`.

Проверка доступа к API:

```bash
python manage.py shell -c "from shop.moysklad import MoySkladClient; print(MoySkladClient().ping())"
```

### Синхронизация

Ссылки на картинки попадают в БД при синхронизации. Запустите:

```bash
python manage.py sync_moysklad_site_products
```

Если у товара в МойСклад нет изображений, `external_image_url` останется пустым.

### Логи при запросе картинки

Запустите `python manage.py runserver`, откройте каталог в браузере. В консоли смотрите строки с `product_image_proxy` — там причина (нет ссылки, ошибка скачивания и т.д.).

### Сеть

Сервер должен иметь доступ к api.moysklad.ru и к хранилищу МойСклад (в т.ч. порт 8080 по документации). При блокировке скачивание картинок будет падать.

## 3. Почему приходит GIF 0.4 kB

Эндпоинт отдаёт реальное изображение (JPEG/PNG), если скачивание с МойСклад успешно. При любой ошибке возвращается прозрачный 1×1 GIF, чтобы не ломать запрос из `<img>`. Маленький GIF = запасной ответ при ошибке, а не конвертация фото.

## 4. Ошибка placehold.co (ERR_NETWORK_CHANGED)

Это уже запрос к внешнему плейсхолдеру: бэкенд вернул «нет картинки», фронт подставил placehold.co, и он не загрузился из‑за сети. Сначала устраните причину на бэкенде (п. 1–2).
