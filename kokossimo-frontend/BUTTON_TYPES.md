# Button Types

Рабочая классификация кнопок во frontend Kokossimo.

Эти коды можно использовать в дальнейших задачах в короткой форме:
- "сделай эту кнопку типом `Action Solid`"
- "сделай эту кнопку типом `A1`"
- "замени здесь на `B1`"
- "нужен степпер типа `S1`"

## Базовые принципы

- Буква обозначает семейство:
  - `A` — основные action-кнопки
  - `B` — вторичные/мягкие action-кнопки
  - `C` — текстовые действия
  - `I` — иконковые кнопки
  - `S` — кнопки stepper/quantity control
- Цифра обозначает конкретный визуальный паттерн.
- Если у кнопки есть особое состояние, его называем отдельно:
  - `default`
  - `hover`
  - `disabled`
  - `active`

## A1 — Action Solid

Главная целевая кнопка действия.

Визуально:
- фон: тёмный бордовый / винный
- текст: белый
- рамка: отсутствует или в цвет фона
- высота: обычно `40px`, иногда `46-48px`
- скругление: `4px`
- текст: uppercase

Состояния:
- `default`: тёмный бордовый фон
- `hover`: фон темнее
- `disabled`: taupe-фон, приглушённый текст, без обводки, `cursor: not-allowed`

Текущие примеры в проекте:
- `cart-summary__submit`
- `cart-empty__button`
- `auth-btn auth-btn--primary`
- `profile-btn profile-btn--primary`
- `profile-btn--repeat`
- `profile-btn--to-cart`
- `payment-primary-button--active`

## A2 — Primary Outline

Основная кнопка без заливки, но с явной рамкой.

Визуально:
- фон: прозрачный
- текст: бордовый / accent
- рамка: бордовая
- высота: обычно `40-42px`
- скругление: `4px`
- текст: uppercase

Состояния:
- `default`: прозрачный фон
- `hover`: заливка accent, текст белый
- `disabled`: светлая рамка, светлый текст, без hover-эффекта

Текущие примеры:
- `product-card__btn`
- `cart-items__clear`
- `payment-secondary-button`
- `btn-secondary`
- `auth-btn--ghost`

## B1 — Soft Outline

Мягкая вторичная кнопка с менее контрастной рамкой.

Визуально:
- фон: белый или прозрачный
- текст: приглушённый красно-коричневый / розовато-бордовый
- рамка: светлее, чем у `A2`
- высота: обычно `40px`
- скругление: `4px`

Состояния:
- `default`: мягкий outline
- `hover`: может заполняться акцентным цветом или чуть затемняться
- `disabled`: очень светлая рамка и текст

Текущие примеры:
- `profile-btn--outline`
- `btn-secondary--outline`

## B2 — Support Outline

Поддерживающая outline-кнопка с мягким бежево-песочным акцентом.

Визуально:
- фон: прозрачный
- текст: `#B4928A`
- рамка: `#B4928A`
- высота: обычно `40px`
- скругление: `4px`

Состояния:
- `default`: прозрачный фон, текст и рамка `#B4928A`
- `hover`: фон и рамка `#B4928A`, текст `#6C5550`
- `disabled`: прозрачный фон, текст и рамка `#C5B8B5`

Текущие примеры:
- `profile-btn--support`

## C1 — Text Action

Текстовая кнопка без рамки и без заливки.

Визуально:
- фон: none
- рамка: none
- текст: muted / accent
- используется как вспомогательное действие

Состояния:
- `default`: обычный текстовый цвет
- `hover`: затемнение или подчёркивание
- `disabled`: снижение opacity

Текущие примеры:
- `auth-link`
- ссылки формата `btn-link` и аналогичные inline-action элементы

## I1 — Icon Utility

Иконковая кнопка интерфейса без явной плашки.

Визуально:
- квадратная зона клика
- фон: прозрачный
- внутри только иконка

Состояния:
- `default`: иконка в accent
- `hover`: затемнение / смена иконки
- `active`: допускается badge, filled heart, selected state

Текущие примеры:
- `header__icon-btn`
- `product-card__favorite`
- `auth-password-toggle`

## S1 — Stepper Circle

Кнопки плюс/минус в корзине.

Визуально:
- форма: круг
- фон: прозрачный
- рамка: accent
- размер: компактный

Состояния:
- `default`: outline-circle
- `hover`: accent fill + white icon
- `disabled`: lower opacity

Текущие примеры:
- `.cart-item__quantity .quantity-btn`

## S2 — Stepper Square

Кнопки плюс/минус в карточке товара.

Визуально:
- форма: квадрат / rounded square
- фон: прозрачный
- рамка: accent
- размер: заметно крупнее `S1`

Состояния:
- `default`: outline-square
- `hover`: accent fill + white icon
- `disabled`: стандартное приглушение

Текущие примеры:
- `product-card__qty-btn`

## D1 — Disabled Solid

Отдельный служебный тип для заблокированных primary-кнопок, когда важно явно показать недоступность действия.

Визуально:
- светлый фон
- светлая рамка или отсутствие контраста
- приглушённый текст
- `cursor: not-allowed`

Текущие примеры:
- `payment-primary-button` без `--active`
- `profile-btn--disabled`
- disabled-состояния `profile-btn--primary`

## Как использовать дальше

Если в следующей задаче ты говоришь:
- "эта кнопка должна быть `Action Solid`" — я делаю primary solid
- "эта кнопка должна быть `A1`" — я делаю primary solid
- "эта кнопка должна быть `A2`" — я делаю accent outline
- "эта кнопка должна быть `B1`" — я делаю мягкий outline
- "здесь нужен `C1`" — я делаю текстовое действие
- "здесь нужен `I1`" — я делаю иконковую utility-кнопку
- "здесь нужен `S1` / `S2`" — я делаю нужный вариант stepper-кнопки

## Замечание

Сейчас эти типы уже существуют в проекте, но реализованы разрозненно по страницам.
Если захотим, следующим шагом можно вынести их в единый слой:
- CSS custom properties
- shared utility classes
- единый React `Button` component
